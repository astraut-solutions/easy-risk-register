# Accessibility Statement for Easy Risk Register

**Effective Date:** November 15, 2025

## Our Commitment to Accessibility

Easy Risk Register is committed to ensuring digital accessibility for people with disabilities. We continually improve the user experience for everyone and work toward meeting the Web Content Accessibility Guidelines (WCAG) 2.1 AA.

## Measures to Support Accessibility

Easy Risk Register takes the following measures to support accessibility:

- Include accessibility as part of our design and development process
- Integrate automated accessibility testing in our development workflow
- Perform regular accessibility reviews and testing

## Conformance Status

Easy Risk Register targets **WCAG 2.1 AA** conformance. Testing includes automated tools (axe-core) and manual review; conformance is assessed on an ongoing basis as the application evolves.

## Technical Specifications

The application uses the following technologies:

- HTML5
- CSS3
- JavaScript (ES6+)
- React
- TypeScript
- Tailwind CSS

## Known Limitations

While we strive to maintain WCAG 2.1 AA alignment, we acknowledge:

- Some complex visualizations may require additional description for screen readers
- Dynamic content updates are continuously being improved for screen reader compatibility

## Feedback and Contact Information

We welcome feedback on the accessibility of Easy Risk Register.

- GitHub Issues: https://github.com/easy-risk-register/easy-risk-register/issues

## Assessment Approach

Easy Risk Register is assessed using:

- Automated testing tools (axe-core)
- Manual keyboard navigation testing
- Screen reader testing (NVDA, JAWS, VoiceOver)
- Color contrast analysis
- Focus indicator visibility testing

## Formal Review

This accessibility statement was created on November 15, 2025 and should be reviewed periodically based on application changes.
